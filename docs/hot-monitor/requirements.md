# 热点监控系统 — 需求文档

## 项目背景

AI 编程博主需要第一时间获取 AI 领域的最新热点（如大模型更新、工具发布等），不能依赖人工手动搜索，要利用自动化工具发现热点变化并及时通知，走在"吃瓜第一线"。

## 核心功能

### 1. 关键词监控
- 用户手动输入要监控的关键词（如 "GPT-5"、"Claude 4"、"DeepSeek"）
- 系统定时从多个数据源搜索该关键词相关内容
- 利用 DeepSeek AI 识别假冒/不相关/标题党内容
- 确认真实热点后第一时间推送通知

### 2. 热点自动发现
- 每隔 **30 分钟**自动搜集用户指定范围（如 "AI 编程"、"大模型"）内的热点
- 多信息源聚合，避免单一信息源偏差
- AI 评分排序后展示

### 3. 多信息源
| 信息源 | 方式 | 说明 |
|--------|------|------|
| Twitter (X) | TwitterAPI.io | 搜索推文、获取趋势 |
| 网页搜索 | 后端爬虫 | 直接爬取搜索引擎结果页 |

### 4. 通知方式
- **浏览器推送通知**（Web Push API / Notification API）
- **邮件通知**（SMTP）

### 5. 产品形态
- 响应式 Web 页面（桌面 + 移动端）
- Agent Skills 封装（供其他 AI 调用）

## 非功能需求
- 前端设计独特，不千篇一律
- 轻量级，敏捷开发
- 先完成 Web 版，再开发 Agent Skills
